# UnitTesting

## 说明

为新增或修改代码补充单元测试或等价验证逻辑。

## 适用场景

- 新功能
- 缺陷修复
- 风险较高的逻辑变更

## 输入

- 代码变更
- 业务规则
- 关键边界
- 运行环境

## 输出

- 测试用例
- 测试代码
- 验证说明

## 关键规则

- 覆盖正常、边界和异常场景
- 回归测试优先覆盖历史风险
- 测试命名应能反映业务意图

## 必填项

- 测试用例
- 测试代码
- 验证说明

## 输出要求

- 必须覆盖关键路径
- 必须体现回归风险点
- 必须能直接用于验证变更
