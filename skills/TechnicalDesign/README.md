# TechnicalDesign

## 说明

生成技术设计文档，描述系统方案、模块拆分、调用链、数据流和边界条件。

## 适用场景

- 需求已明确，需要落地方案
- 需要评估模块拆分
- 需要确认数据库、缓存和接口设计

## 输入

- 需求分析结果
- 项目上下文
- 运行环境
- 现有架构约束

## 输出

- 总体方案
- 模块设计
- 调用链
- 数据设计
- 缓存设计
- 异常处理

## 关键规则

- 设计必须可实现
- 设计必须兼容当前版本
- 设计必须说明风险和边界

## 必填项

- 总体方案
- 模块设计
- 调用链
- 数据设计
- 缓存设计
- 异常处理

## 输出要求

- 设计必须能指导实现
- 必须说明与现有系统的兼容关系
- 必须说明关键风险和边界条件
