# CodeGeneration

## 说明

根据现有标准、运行环境和设计文档生成最小可用代码修改。

## 适用场景

- 实现新功能
- 按方案修改现有逻辑
- 生成样板代码

## 输入

- 技术方案
- 任务拆分
- 代码上下文
- 运行环境

## 输出

- 代码变更
- 必要注释
- 基础自检说明

## 关键规则

- 只做当前需求所需的最小改动
- 不能越过运行环境限制
- 不能忽略项目现有写法

## 必填项

- 代码变更
- 必要注释
- 基础自检说明

## 输出要求

- 代码必须符合当前 Profile
- 注释必须准确且简洁
- 变更范围必须尽量小
