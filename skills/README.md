# Skills

## 说明

Skills 是最小原子能力层，用于把需求分析、设计、实现、测试、审查和文档输出拆成可复用模块。

## 当前能力

- `RequirementAnalysis`
- `TechnicalDesign`
- `RiskAnalysis`
- `ExistingLogicAnalysis`
- `DatabaseDesign`
- `ApiDesign`
- `TaskBreakdown`
- `CodeGeneration`
- `FrontendImplementation`
- `Debugging`
- `UnitTesting`
- `CodeReview`
- `ApiDocumentation`
- `ChangeSummary`

## 使用原则

- 一个 Skill 只负责一类任务
- Skill 输出必须稳定
- Skill 应该能被多个 Workflow 复用
- Skill 不应包含不必要的流程编排

## 常见选择

- 确认需求：`RequirementAnalysis`
- 确认现有逻辑：`ExistingLogicAnalysis`
- 排查问题：`Debugging`
- 修改代码：`CodeGeneration`
- 修改前端页面：`FrontendImplementation`
- 输出文档：`ApiDocumentation` 或 `ChangeSummary`
