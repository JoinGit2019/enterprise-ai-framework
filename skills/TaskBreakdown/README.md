# TaskBreakdown

## 说明

将需求拆解为可执行任务。

## 适用场景

- 开发前排期
- 方案确认后拆分任务
- 大需求拆分成多步交付

## 输入

- 需求
- 技术设计
- 风险分析

## 输出

- 任务列表
- 依赖关系
- 优先级
- 验收点

## 关键规则

- 任务必须可执行
- 任务必须能验收
- 任务之间的依赖关系要明确

## 必填项

- 任务列表
- 依赖关系
- 优先级
- 验收点

## 输出要求

- 每个任务应能单独执行
- 每个任务应有明确完成标准
- 任务拆分不能遗漏关键文档或测试
