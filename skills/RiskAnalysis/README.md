# RiskAnalysis

## 说明

识别需求和方案中可能出现的问题，并给出缓解建议。

## 适用场景

- 新功能上线前
- 数据迁移前
- 性能优化前
- 接口变更前

## 输入

- 需求
- 技术方案
- 运行环境
- 历史问题

## 输出

- 风险项
- 影响范围
- 严重程度
- 缓解方案
- 回滚建议

## 关键规则

- 风险必须具体
- 风险必须能落地到措施
- 不能只写抽象结论

## 必填项

- 风险项
- 影响范围
- 严重程度
- 缓解方案
- 回滚建议

## 输出要求

- 风险要对应具体场景
- 每个风险都应有缓解动作
- 必要时标记是否需要人工确认
