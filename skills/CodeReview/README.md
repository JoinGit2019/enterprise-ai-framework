# CodeReview

## 说明

审查代码变更，关注兼容性、可维护性、安全性和性能。

## 适用场景

- 合并前审查
- 需求实现后自检
- 线上修复复核

## 输入

- 代码差异
- 需求背景
- 运行环境
- 相关文档

## 输出

- 问题清单
- 风险清单
- 可接受项
- 修改建议

## 关键规则

- 先审查兼容性
- 再审查安全性
- 再审查性能与可维护性

## 必填项

- 问题清单
- 风险清单
- 可接受项
- 修改建议

## 输出要求

- 结论必须清晰
- 问题必须可定位
- 建议必须可执行
