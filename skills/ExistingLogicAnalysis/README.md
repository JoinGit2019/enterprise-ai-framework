# ExistingLogicAnalysis

## 说明

用于阅读现有项目代码，确认某个功能的真实逻辑，或排查问题前先梳理现有实现与调用链。

这个 Skill 的重点不是立即修改代码，而是先把事实确认清楚，并输出可追溯的逻辑说明文档。

## 适用场景

- 确认现有功能是如何实现的
- 梳理某个接口、任务、定时脚本或业务流程的调用链
- 排查问题前先确认实际逻辑
- 判断现有逻辑是否符合业务预期
- 给产品、测试或研发输出功能说明

## 输入

- 功能名称或问题描述
- 相关接口、页面、任务或模块线索
- 代码上下文
- 日志、数据样例或复现步骤
- 当前项目结构缓存

## 输出

- 功能或问题概述
- 代码入口
- 调用链
- 关键业务规则
- 数据流
- 状态流转
- 外部依赖
- 已确认事实
- 疑点与待确认事项
- 结论与建议

## 关键规则

- 先确认事实，再判断问题
- 必须引用实际代码入口和调用链
- 不把猜测写成结论
- 不在未确认逻辑前直接修改代码
- 区分“当前实现如此”和“业务应该如此”

## 必填项

- 功能或问题概述
- 代码入口
- 调用链
- 关键业务规则
- 已确认事实
- 疑点与待确认事项
- 结论与建议

## 输出要求

- 必须能让没有读过代码的人理解当前逻辑
- 必须说明结论依据来自哪里
- 必须标记无法确认的部分
- 如果发现疑似问题，必须说明触发条件和影响范围
