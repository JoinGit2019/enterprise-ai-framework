# RequirementAnalysis

## 说明

将用户描述、产品文档或一句话目标转换为可执行的需求分析文档。

## 适用场景

- 新增功能
- 变更现有流程
- 梳理产品想法
- 明确实现边界

## 输入

- 需求描述
- 现有系统上下文
- 业务约束
- 版本环境

## 输出

- 背景
- 目标
- 范围
- 业务流程
- 影响范围
- 风险
- 待确认事项

## 关键规则

- 先澄清再实现
- 不把猜测写成事实
- 明确说明不包含的范围

## 必填项

- 背景
- 目标
- 范围
- 业务流程
- 影响范围
- 风险
- 待确认事项

## 输出要求

- 必须能直接进入技术设计
- 必须列出待确认问题
- 必须明确边界和不包含项
