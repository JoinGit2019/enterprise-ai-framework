# FrontendImplementation

## 说明

用于实现或修改前端页面、模板、组件、样式和 JavaScript 交互。

## 适用场景

- 修改后台页面
- 修改普通前端页面
- 新增表单字段
- 调整列表展示
- 修改按钮交互
- 调整 CSS 样式
- 修改 Ajax 请求
- 修复页面 JS 问题

## 输入

- 页面或模板文件
- 组件文件
- 相关 CSS 文件
- 相关 JavaScript 文件
- 接口、状态或模板变量
- 现有页面风格

## 输出

- HTML / 模板变更
- 组件变更
- CSS 变更
- JavaScript 变更
- 交互说明
- 验证说明

## 关键规则

- 先确认页面入口和依赖文件
- 先查找现有页面、组件、样式和工具函数
- 优先复用已有实现
- 不破坏模板变量、组件状态、权限逻辑和渲染条件
- 不随意引入新依赖
- 样式修改必须控制影响范围
- 样式优先放在 CSS、样式文件或项目既有样式体系中，不随意写在 HTML 内联样式里
- JS 修改必须确认接口参数和返回结构
- 必须考虑当前项目的浏览器目标和构建环境兼容性

## 必填项

- 变更位置
- 影响范围
- 交互逻辑
- 验证方式

## 输出要求

- 必须说明修改了哪些页面或资源
- 必须说明是否影响现有接口
- 必须说明如何验证页面行为
- 必须说明是否复用了现有组件、样式或工具函数
