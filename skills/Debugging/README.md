# Debugging

## 说明

用于分析错误、定位根因并给出修复建议。

## 适用场景

- 线上报错
- 本地复现异常
- 逻辑结果不正确
- 性能异常

## 输入

- 错误信息
- 日志
- 代码上下文
- 复现步骤

## 输出

- 根因分析
- 修复方案
- 验证步骤

## 关键规则

- 先定位根因再改代码
- 不能只修表象
- 不能忽略复现路径

## 必填项

- 根因分析
- 修复方案
- 验证步骤

## 输出要求

- 必须给出可复现链路
- 必须说明根因与现象的关系
- 必须说明验证方式
