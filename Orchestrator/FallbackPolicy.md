# FallbackPolicy

## 说明

当上下文识别不充分或存在冲突时，编排器必须进入兜底策略，而不是盲目继续。

## 何时触发

- PHP 版本无法确认
- ThinkPHP 版本无法确认
- 项目类型无法确认
- 用户目标存在歧义
- 需求不足以直接路由到工作流

## 处理规则

- 优先询问用户
- 优先展示有限选项
- 不默认选择最新版本
- 不默认选择最激进方案
- 不默认进行破坏性变更

## 询问方式

问题必须短、明确、可选。

例如：

- 当前项目是 `PHP7.2-ThinkPHP5.1` 还是 `PHP7.4-ThinkPHP5.1`？
- 这次任务更偏向功能新增还是缺陷修复？
- 需要先输出需求分析文档还是直接进入技术设计？

## 允许继续的情况

在不影响正确性的前提下，可以继续的内容包括：

- 生成分析框架草稿
- 输出待确认问题清单
- 输出风险假设
- 输出文档模板

## 禁止行为

- 不允许在版本不明时直接生成高版本语法
- 不允许在框架不明时直接使用框架专属 API
- 不允许把猜测当成事实写入设计文档
