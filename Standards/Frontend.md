# Frontend

## 说明

这里定义前端代码的通用规则，包括 HTML、CSS、JavaScript、模板页面和组件页面。

## 规则

- 先确认前端代码属于模板页面、静态页面、组件源码还是构建产物
- 先查找现有页面、组件、样式、工具函数和交互逻辑
- 优先复用已有实现，不重复造轮子
- 不直接修改构建后的压缩文件，除非项目只有该文件可维护
- 不随意引入新框架、新插件或构建工具
- 修改 HTML 或模板时检查变量、权限判断、循环结构和已有选择器
- 修改 CSS 时控制作用域，避免污染全局样式
- 样式优先写在 CSS、样式文件或项目既有样式体系中，不随意写在 HTML 的 `style` 属性里
- 不为了局部需求新增大段全局 CSS
- 修改 JavaScript 时检查事件绑定、接口参数、返回结构和异常提示
- 前端改动必须考虑浏览器兼容、构建环境兼容和已有页面风格
- 不使用当前项目构建链或浏览器目标不支持的语法
- 新增交互必须考虑空数据、加载中、失败提示和重复点击
